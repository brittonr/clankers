# Evidence: command prompt rejection and event emission parity

## Summary

- Cover allowed prompt request shell work and concurrent-prompt rejection through `crates/clankers-controller/src/command.rs`.
- Assert `CoreInput::PromptRequested` yields ordered reducer output `BusyChanged(true)` then `StartPrompt { effect_id, prompt_text, image_count }` for an allowed prompt.
- Assert `SessionController::handle_command(SessionCommand::Prompt { .. })` consumes that reducer plan, forwards the prompt to the provider shell seam, and preserves `[DaemonEvent::AgentStart, DaemonEvent::PromptDone { error: None }]` with no duplicate acknowledgement event.
- Assert concurrent prompt start preserves unchanged state, preserves the error `DaemonEvent::SystemMessage` category, and matches typed `CoreError::Busy` with no shell work.
