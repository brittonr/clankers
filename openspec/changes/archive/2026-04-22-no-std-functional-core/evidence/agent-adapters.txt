# Evidence: agent adapter parity

## Summary

- `crates/clankers-agent/src/lib.rs` now exposes explicit shell helpers `Agent::apply_controller_thinking_level(...)` and `Agent::apply_core_filtered_tools(...)` so controller-owned reducer effects are applied through agent adapter methods without pulling `clankers-core` into the agent's runtime dependency graph.
- `crates/clankers-agent/src/turn/mod.rs` now names the authoritative input `controller_tools` and builds turn tool definitions through `tool_definitions_from_controller_inventory(...)`, making the controller-provided filtered inventory the only turn-time tool source for the migrated slice.
- `crates/clankers-controller/src/core_effects.rs` translates `CoreThinkingLevel` into `clankers_provider::ThinkingLevel` before crossing into `clankers-agent`, keeping the agent runtime boundary shell-native while still consuming reducer-produced thinking effects.
- `crates/clankers-agent/src/turn/execution.rs` now threads that same `controller_tools` inventory through tool lookup, so removed tools fail as `Tool 'direct_tool' not found` instead of being recovered from hidden turn-local state.
- The remaining user allowlist checks stay turn-local by design and remain covered separately as a shell-only capability rail.
