## ADDED Requirements

### Requirement: Provider/router boundary policy has one owner [r[remaining-coupling-drain.provider-router-boundary-collapse]]

Provider compatibility layers MUST converge so request shaping, auth, discovery, routing, retry, cache-key, and stream-normalization policy each have one declared owner, while adapters translate DTOs, streams, and errors only.

#### Scenario: Concern owner map is complete [r[remaining-coupling-drain.provider-router-boundary-collapse.owner-map]]
- GIVEN provider/router code owns body shaping, discovery, auth refresh or probing, routing, fallback, cooldown, retry, cache-key, stream normalization, or error mapping
- WHEN boundary validation inventories provider concerns
- THEN every concern MUST name exactly one owner module or backend entrypoint
- AND compatibility adapters MUST name the bridge or backend helper they delegate to

#### Scenario: Compatibility adapters stay thin [r[remaining-coupling-drain.provider-router-boundary-collapse.thin-adapters]]
- GIVEN `clankers-provider` compatibility code calls `clanker-router` or routed backends
- WHEN request, stream, cache-key, auth, retry, or discovery behavior changes
- THEN compatibility adapters MUST translate neutral/provider DTOs, stream events, metadata, and errors only
- AND they MUST NOT duplicate provider-native body construction, auth probe, retry, cooldown, fallback, or stream state-machine policy

#### Scenario: Duplicate request DTOs are collapsed or rail-checked [r[remaining-coupling-drain.provider-router-boundary-collapse.duplicate-dtos]]
- GIVEN local provider and router request/event DTOs both remain public
- WHEN a field or constructor is added
- THEN constructor-count and shared-field serde parity rails MUST prove every compatibility projection is updated
- AND any duplicate field set MUST have a documented convergence condition or be collapsed into a shared owner

#### Scenario: Fixtures prove adapter delegation [r[remaining-coupling-drain.provider-router-boundary-collapse.fixtures]]
- GIVEN compatibility adapters translate representative histories
- WHEN fixture tests run
- THEN branch summaries, compaction summaries, tool replay, reasoning, metadata, cache inputs, and retry/refresh inputs MUST match explicit expected fixtures
- AND the expected fixtures MUST NOT be generated by calling the body builder under test

#### Scenario: Provider/router validation runs [r[remaining-coupling-drain.provider-router-boundary-collapse.validation]]
- GIVEN provider/router boundary changes are implemented
- WHEN focused validation runs
- THEN responsibility rails, request-shape fixtures, constructor-count parity tests, and fake-routed-backend adapter tests MUST pass

#### Scenario: Provider/router closeout is gated [r[remaining-coupling-drain.provider-router-boundary-collapse.closeout]]
- GIVEN the provider/router boundary change is ready to close
- WHEN closeout validation runs
- THEN affected cargo checks, Cairn gates, Cairn validation, and diff checks MUST pass
