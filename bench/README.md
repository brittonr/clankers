# Token Usage Benchmark: pi vs clankers

A/B comparison of token consumption across identical prompts.

## Quick Start

```bash
# Full run — all 10 prompts, both agents
./bench/token-bench.sh

# Single prompt (fast sanity check)
./bench/token-bench.sh --prompt 10 --no-thinking

# One suite only
./bench/token-bench.sh --suite read-only

# Pin model, repeat 3 times for variance
./bench/token-bench.sh --model claude-sonnet-4-20250514 --runs 3

# Only measure pi
./bench/token-bench.sh --pi-only

# Run against a different repo
./bench/token-bench.sh --workdir ~/git/other-project
```

## What It Measures

Each prompt runs through both agents in headless (non-interactive) mode with
tools, skills, extensions, and session persistence disabled. The script
captures:

| Metric | Description |
|--------|-------------|
| **Turns** | Number of LLM round-trips (user→assistant) |
| **Input tokens** | Tokens sent to the model (system prompt + conversation) |
| **Output tokens** | Tokens generated by the model |
| **Cache read** | Tokens served from prompt cache |
| **Cache write** | Tokens written to prompt cache |
| **Total tokens** | Input + output |
| **Est. cost** | Estimated USD (pi only — clankers doesn't embed pricing) |

## Prompt Suites

| Suite | Tests |
|-------|-------|
| `read-only` | File reading, simple extraction |
| `search` | Codebase search via grep/find |
| `edit` | File modification + cleanup |
| `multi-tool` | Chained tool calls |
| `reasoning` | No-tool analysis of code structure |
| `no-tools` | Pure generation, no tool access |

## Controls

The benchmark disables these to isolate the agent core from addons:

- **Skills** (`--no-skills` / not loaded)
- **Extensions** (`--no-extensions`)
- **Prompt templates** (`--no-prompt-templates`)
- **Session persistence** (`--no-session`)
- **Thinking** (off by default; enable with `--thinking high`)

Both agents use the same model, same working directory, and same prompt
text. The main differences being measured are:

1. **System prompt size** — pi and clankers have different default system prompts
2. **Tool definitions** — different tool schemas = different input token counts
3. **Tool call patterns** — how many round-trips each agent needs
4. **Output verbosity** — how much text each agent generates

## Output

Results go to `/tmp/token-bench/<timestamp>/`. Each prompt produces:

- `NNN_rN_pi_raw.jsonl` — raw pi JSON event stream
- `NNN_rN_pi_usage.json` — extracted usage summary
- `NNN_rN_ck_raw.jsonl` — raw clankers JSON event stream
- `NNN_rN_ck_usage.json` — extracted usage summary
- `summary.json` — grand totals

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `CLANKERS_BIN` | `target/debug/clankers` | Path to clankers binary |
| `PI_BIN` | `pi` | Path to pi binary |
| `BENCH_MODEL` | `claude-sonnet-4-20250514` | Model for both agents |
| `BENCH_WORKDIR` | repo root | Working directory |
